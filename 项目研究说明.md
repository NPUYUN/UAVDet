# UAVDet 项目研究说明

## 1. 项目相关研究现状及发展动态

近年来，低空经济快速发展，无人机在电力巡检、城市安防、应急救援、交通监管等场景中的应用显著增加，带动了“低空目标智能感知”研究热度持续上升。与传统通用目标检测任务相比，无人机目标检测面临小目标占比高、背景干扰强、遮挡与光照变化频繁、目标运动速度快等特殊挑战。当前研究总体呈现三条并行发展趋势：第一，检测算法轻量化与高精度协同优化，典型方向是将 YOLO 系列与轻量骨干网络、注意力机制、重参数化结构融合，以满足边缘端实时部署需求；第二，多尺度特征建模持续增强，研究者通过跨层特征反馈、感受野扩展、特征重校准等机制提升微小目标检出率；第三，系统化与工程化能力成为评价重点，单一模型性能已不足以覆盖复杂任务需求，多模型融合、任务调度、可视化回溯、在线更新等能力逐步成为实用系统标配。同时，数据问题仍是瓶颈，真实场景采集和标注成本高，促使仿真数据生成与真实-仿真联合训练成为新动态。总体看，当前领域正从“单点算法优化”向“算法-数据-系统一体化”演进。

## 2. 项目实施的目的、意义

本项目实施的核心目的，是构建一套面向低空复杂场景的无人机目标检测技术体系，在“轻量化、精度、实时性、可部署性”之间形成可落地的综合平衡。具体而言，项目通过 RepViT-Drone、HHFNet 以及多模型融合系统的联合设计，解决单模型在复杂背景下稳定性不足、边缘设备部署受限、不同任务优先级难统一等现实问题，最终形成可用于巡检、安防、应急等任务的检测能力。

项目意义主要体现在三个层面。其一，学术层面：围绕小目标检测、多尺度特征增强、模型轻量化等关键议题提供可复现的研究路径，为后续改进提供结构化基础。其二，工程层面：通过模块化、多线程和融合推理机制提升系统稳定性与可维护性，推动“论文方法”向“工程系统”转化。其三，应用层面：在低空治理与行业智能化背景下，项目能够提升目标发现效率与风险预警能力，降低人工巡查成本，具有明显的实际价值和推广潜力。整体上，该项目兼顾研究深度和应用导向，符合当前智能感知技术发展与产业需求。

## 3. 项目研究内容和拟解决的关键问题

本项目围绕无人机目标检测“算法、数据、系统”三条主线开展研究。第一条主线是轻量化高性能检测算法 RepViT-Drone，基于 YOLOv11-N 引入 RepViTBlock、通道注意力下采样、RIC 与 AFCA 等结构，在保证模型体积较小（约 1.8MB~1.9MB）的同时提升特征表达与细节恢复能力，重点优化边缘端实时检测中的精度损失问题。第二条主线是 HHFNet 多尺度特征增强网络，利用 CSF 与 PRFA 强化深浅层协同和渐进式感受野聚合，针对复杂背景、尺度变化大、微小目标难检出等问题提升鲁棒性。第三条主线是多模型融合检测系统，通过 WBF 与 SBF 两类融合策略，配合输入调度、融合推理、结果回溯与数据管理模块，实现不同任务目标（精度优先或召回优先）下的自适应应用。

拟解决的关键问题主要包括：1）轻量模型在小目标场景中表征能力不足，导致漏检率高；2）复杂背景和快速运动条件下检测稳定性不够，模型输出波动大；3）单模型在不同业务优先级下适配能力有限，难兼顾误检控制与召回需求；4）真实数据采集与标注成本高，数据分布覆盖不足影响泛化性能；5）算法与系统脱节，导致实验指标良好但工程落地困难。项目通过模型结构改造、多尺度增强、融合推理与仿真数据支撑协同解决以上问题，形成可复用的无人机检测研究范式。

## 4. 项目研究与实施的基础条件

从现有材料看，本项目已具备较完整的研究基础与实施条件。首先，在研究框架方面，项目目标清晰，已明确“轻量化 + 高精度 + 实时性”的总体定位，并形成 RepViT-Drone、HHFNet 与多模型融合系统三条互补研究主线，避免了单点突破导致的瓶颈。其次，在方法储备方面，项目已经沉淀了较成熟的关键模块设计思路，包括 RepViTBlock、通道注意力下采样、RIC、AFCA、CSF、PRFA 以及 WBF/SBF 融合策略，说明项目并非停留在概念阶段，而是具备可执行的技术方案。

在数据与评估条件方面，项目已明确采用 DUT Anti-UAV 与 Det-Fly 两类具有代表性的无人机检测数据集，覆盖多场景、多光照与微小目标挑战；同时已建立统一输入分辨率、常规增强流程与包含 Precision、Recall、mAP50、mAP50:95、Latency、FPS、Model Size 在内的评价指标体系，具备系统对比与量化分析基础。在工程实现条件方面，项目提出模块化、多线程系统架构，并包含任务调度、结果展示、历史回溯与数据管理能力，具有较强的工程可实施性。

此外，项目文档基础较好，README 已形成研究背景、方法概览、结论与创新点的完整叙述，并保留了后续环境配置、训练命令、推理示例等扩展接口，利于团队协作和阶段复盘。总体上，项目已具备“理论方向明确、方法路径可落地、评估体系可验证、系统框架可扩展”的实施基础条件。

## 5. 项目实施方案

本项目实施方案按照“需求分析—算法研发—数据支撑—系统集成—测试优化—成果固化”六步推进。第一阶段开展需求分析与任务定义，明确应用场景中对实时性、精度、误检率、部署资源的约束，建立统一实验协议与指标口径，确保后续结果可比、可复现。第二阶段推进算法研发：在 YOLOv11-N 基线上实现 RepViT-Drone 结构改造，分模块验证 RepViTBlock、RIC、AFCA、通道注意力下采样的独立贡献；并同步训练 HHFNet，重点评估 CSF 与 PRFA 对多尺度和复杂背景场景的提升效果。

第三阶段开展数据支撑工作，基于现有真实数据集进行清洗、重采样与增强策略优化，同时构建仿真数据生成流程，补足极端光照、遮挡、远距微小目标等长尾样本，形成“真实+仿真”联合训练与验证机制。第四阶段进行系统集成与融合推理，将单模型推理服务化，建立多线程调度、融合策略切换（WBF/SBF）、结果可视化与历史回溯模块，支持不同任务模式下快速切换。第五阶段执行多轮测试优化，包括离线精度评估、在线时延测试、稳定性压力测试与场景化对比测试，针对瓶颈进行模型剪枝、参数重整或策略调优。第六阶段完成成果固化，输出模型版本、系统文档、实验报告与可复现实验脚本。通过上述分阶段实施，确保项目既能产出可解释的研究结论，也能形成可运行的工程系统。

## 6. 项目的技术路线、创新点与项目特色

项目技术路线采用“基础模型构建—结构增强—多尺度协同—融合决策—系统落地”的递进式方案。首先，以 YOLOv11-N 为轻量基线建立初始检测能力；其次，围绕轻量模型表达不足问题引入 RepViT-Drone 关键模块，增强跨通道与跨区域信息交互，同时通过 RIC 与 AFCA 改善上采样细节恢复和融合后特征校准；再次，构建 HHFNet，通过 CSF 与 PRFA 建立深浅层反馈与感受野渐进聚合，提升对多尺度和复杂背景的适应性；随后在系统层以 WBF/SBF 实现多模型融合决策，依据任务目标在定位质量与选择性精度之间动态权衡；最终通过模块化、多线程架构实现实时部署与可维护运行。

创新点体现在三方面：一是算法创新，将 RepViT 思路与 YOLO 轻量检测框架深度结合，并配套细节恢复与自适应校准机制，兼顾模型尺寸与检测性能；二是多尺度机制创新，利用跨尺度反馈与渐进式感受野聚合强化小目标和复杂场景鲁棒性；三是系统创新，打通模型训练、融合推理、可视化回溯和仿真数据支撑链路，实现研究到应用的闭环。

项目特色在于“一体化”和“可迁移性”：不是单纯追求某一指标最优，而是强调多指标均衡与工程落地；同时融合策略、调度机制和数据生成流程均具有可扩展性，便于迁移到其他低空感知任务或不同硬件平台，具备较高应用推广价值。

## 7. 项目研究已有基础

### 7.1 与本项目有关的研究积累和已取得的成绩

项目已形成较系统的研究积累。第一，已完成低空无人机检测问题分析与技术路线拆解，明确“轻量化、高精度、实时性”三目标协同路径。第二，在算法层面已提出并验证 RepViT-Drone 与 HHFNet 两类核心方案，前者在模型体积与精度平衡方面表现突出，后者在复杂背景与多尺度目标场景下具备更高稳定性。第三，在系统层面已形成多模型融合检测方案，支持 WBF 与 SBF 两种策略，实验结论显示融合方案在误检控制与整体可靠性方面优于单模型。第四，在数据层面已建立以 DUT Anti-UAV、Det-Fly 为核心的训练评估基础，并具备仿真数据生成能力，用于缓解真实数据不足与长尾样本缺失问题。第五，在文档与成果沉淀方面，项目已完成 README 级别的完整研究说明与阶段结论总结，为后续答辩和成果展示打下基础。

### 7.2 已具备的条件、尚缺少的条件及解决方法

已具备条件主要包括：明确的研究主线、可执行的模型设计方案、可用的数据集与评估指标体系、可集成的系统框架思路，以及初步实验结论支撑。这些条件保证项目具备持续推进能力。

尚缺少条件主要体现在三个方面：一是工程化细节文档仍需完善，如环境依赖版本、训练/推理命令、权重管理规范、复现实验脚本；二是更大规模场景化验证不足，特别是针对极端天气、复杂遮挡和跨设备部署的稳健性测试；三是高质量标注数据仍相对紧张，影响跨场景泛化评估深度。

对应解决方法为：第一，建立标准化实验与部署文档模板，补全环境、命令、目录、日志与版本记录；第二，构建分层测试计划，增加离线对比、在线压力、跨平台时延与长期运行稳定性测试；第三，继续推进仿真数据生成并引入真实-仿真混合训练，结合难例挖掘机制定向补充样本；第四，建立模型迭代闭环，通过错误案例回流持续优化网络结构与融合策略。

### 7.3 学校可以提供的条件

学校可提供的支撑条件包括：1）计算资源支持，如实验室 GPU 服务器、共享算力平台及必要的存储环境，满足模型训练与对比实验需求；2）科研指导与学术资源，依托导师团队在目标检测、计算机视觉与工程部署方向的经验，提升研究方案科学性；3）实验平台与协作环境，包括代码托管、版本管理、阶段汇报与成果评估机制，保障项目过程管理；4）论文与文献资源，通过数据库访问与学术讲座帮助团队跟踪前沿动态；5）答辩与成果转化支持，包括中期检查、结题评审、竞赛与成果展示渠道。若后续需要更高算力或实际场景数据采集，学校还可协调校内外联合实验资源，提升项目验证广度与应用价值。

## 8. 项目研究进度安排及各阶段预期成果（中期检查及结题答辩参考）

本项目按“约一年（12个月）”周期组织实施，采用“季度推进 + 里程碑验收”方式，确保中期检查与结题答辩均有明确支撑材料。整体安排如下：

第一阶段（第1-2个月，项目启动与方案细化）：完成研究需求梳理、文献综述、技术路线分解与实验规范制定，明确 RepViT-Drone、HHFNet、多模型融合系统三条主线的接口关系与评估口径。预期成果为开题报告、年度任务分解表、实验指标定义文档、项目管理台账（含风险清单与应对策略）。

第二阶段（第3-4个月，基线构建与算法初版）：完成 YOLOv11-N 基线复现，初步实现 RepViT-Drone 与 HHFNet 的核心模块（如 RepViTBlock、RIC、AFCA、CSF、PRFA），并开展首轮对比与消融实验。预期成果为算法初版代码、阶段实验报告（含 Precision、Recall、mAP50、mAP50:95、Latency、FPS、模型体积指标）、模块有效性结论。

第三阶段（第5-6个月，数据工程与中期检查阶段）：完成数据清洗、标注质检、样本重采样与增强策略优化，推进仿真数据生成流程并建立“真实 + 仿真”联合训练机制；同步完成算法二轮迭代，形成较稳定的阶段模型。该阶段末进行中期检查。预期成果为中期检查报告、数据集版本说明、训练配置文档、阶段模型权重与典型案例分析（含误检/漏检归因）。

第四阶段（第7-9个月，系统集成与融合优化）：完成多模型融合检测系统开发与联调，实现任务调度、单模型/融合推理切换、结果可视化、历史回溯与日志记录功能；对 WBF、SBF 策略进行多场景对比，形成不同任务目标下的参数推荐。预期成果为可运行系统原型、融合策略评估报告、系统接口与部署说明文档、稳定性测试记录。

第五阶段（第10-11个月，性能冲刺与工程化完善）：围绕实时性、鲁棒性和可维护性开展针对性优化，包括推理加速、参数整理、异常场景适配、跨设备部署验证与长时运行测试，完成版本冻结前的综合回归测试。预期成果为最终候选模型与系统版本、完整性能对比数据表、跨场景测试报告、工程化运维手册（含常见问题与排障流程）。

第六阶段（第12个月，成果固化与结题答辩）：完成研究成果归档与论文/技术报告撰写，整理演示材料、答辩文稿和支撑附件，形成“可复现、可展示、可验收”的结题包。预期成果为结题报告、论文定稿（或技术总结报告）、答辩 PPT、演示视频（或现场演示脚本）、项目代码与文档归档清单。

其中，中期检查重点围绕“技术路线执行情况、阶段指标达成度、数据与实验规范性、后续计划可行性”进行评估；结题答辩重点围绕“最终性能指标、创新点证据链、系统可用性与推广价值”进行综合评价，确保项目研究价值与应用价值同步体现。
